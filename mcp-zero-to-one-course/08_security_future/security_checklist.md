# MCP Server 安全性自我檢查清單 (Security Checklist)

在將您的 MCP Server 部署至生產環境或分享給他人之前，請確認以下事項：

## 1. 權限控管
- [ ] **沙盒化**：Server 執行的環境是否與主系統隔離？(例如使用 Docker)。
- [ ] **路徑限制**：檔案操作工具是否限制在特定的工作目錄內？
- [ ] **唯讀優先**：如果不需修改資料，是否使用了 Resources (唯讀) 而非 Tools (讀寫)？

## 2. 輸入驗證
- [ ] **型別檢查**：是否為所有 Tool 參數定義了精確的型別 (int, str, enum)？
- [ ] **長度限制**：是否限制了輸入字串的最大長度，以防記憶體耗盡攻擊？
- [ ] **非法字元**：是否過濾了 shell 注入字元 (如 `;`, `&`, `|`)？

## 3. 日誌與監控
- [ ] **敏感資料**：日誌中是否排除了 API Key、密碼等敏感資訊？
- [ ] **異常偵測**：是否記錄了所有失敗的工具呼叫？

## 4. 使用者體驗與安全
- [ ] **描述清晰**：Tool 的 `description` 是否精確，避免模型誤判而觸發危險操作？
- [ ] **確認機制**：對於毀滅性操作（如刪除資料庫），是否有二次確認邏輯？

---
> **守則**：給予模型最小的權限，進行最大的懷疑。

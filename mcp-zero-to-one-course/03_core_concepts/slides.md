# 第三章：MCP 的三大核心功能 (Core Features)

MCP Server 主要透過三種方式提供功能。理解它們的差異是開發者的必修課。

---

## 1. Tools (工具) 🛠️ —— 「模型的手」

**這是最常用的功能。**

*   **定義**：可執行的函數，模型可以呼叫它來改變世界或獲取資訊。
*   **關鍵點**：模型知道 **如何呼叫** 它（透過 Name, Description, Arguments）。
*   **範例**：
    *   `list_files(path)`：列出檔案。
    *   `send_email(to, subject, body)`：發送郵件。
    *   `calculate_tax(amount)`：計算稅金。

> **一句話總結**：Tools 是給模型「主動」去做的動作。

---

## 2. Resources (資源) 📦 —— 「模型的書」

*   **定義**：唯讀的資料來源，由 Client 提供給模型作為參考背景。
*   **關鍵點**：模型不能「執行」它，但可以「閱讀」它。通常使用 URI 格式（如 `file://`, `postgres://`）。
*   **範例**：
    *   專案文檔 (`docs://v1/api`)。
    *   特定的資料庫紀錄。
    *   即時的系統日誌串流。

> **一句話總結**：Resources 是給模型「被動」參考的資料。

---

## 3. Prompts (提示詞) 📝 —— 「模型的食譜」

*   **定義**：預先定義好的提示詞模板，幫助用戶快速發起特定的任務。
*   **關鍵點**：簡化複雜指令，確保模型輸出高品質結果。
*   **範例**：
    *   `analyze-code`：一段包含「請審查這段代碼的安全性...」的長提示詞。
    *   `summarize-meeting`：自動格式化會議紀錄的模板。

> **一句話總結**：Prompts 是給用戶快速啟動任務的「捷徑」。

---

## 4. 三者對比表

| 特性 | Tools | Resources | Prompts |
| :--- | :--- | :--- | :--- |
| **互動方式** | 模型主動呼叫 | Client 提供給模型 | 用戶選擇使用 |
| **權限** | 可讀/可寫/可執行 | 通常為唯讀 | 僅為文字模板 |
| **主要用途** | 執行動作、獲取即時資料 | 提供靜態/結構化背景 | 導引模型行為 |
| **開發難度** | 中 (需處理參數與邏輯) | 低 (定義路徑與讀取) | 極低 (撰寫模板) |

---

## 5. 實務建議：該選哪一個？

*   如果你要讓 AI **「做事」** (如刪除檔案、發訊息) 👉 **Tools**。
*   如果你要讓 AI **「讀資料」** (如參考文檔、查詢紀錄) 👉 **Resources**。
*   如果你要讓 AI **「按特定格式思考」** (如專業審查、翻譯) 👉 **Prompts**。
